# VocabSynth

事前学習済み自己回帰 Transformer に対して、追加学習なしで仮想トークンを合成する手法の実装。

## 概要

VocabSynth は、構成トークンの埋め込みを組み合わせて新しい複合語を表す仮想トークンを生成する。入力側では構成トークンの埋め込みを静的に合成し、出力側では Contextual Weight Extraction を使って順伝播の隠れ状態から出力重みを取り出す。モデルのパラメータは一切更新しない。

対象は意味が構成要素から導出できる複合語に限る。例えば NaritaCake = Narita + Cake のような地名と食品の組み合わせ。接辞を含む派生語や非合成的な造語は対象外。

## 手法

### 入力側の埋め込み合成

構成トークンの埋め込みを線形結合して仮想トークンの入力埋め込みを作る。合成方法は平均 / 主辞重み付け / 文字長重み付けの3種。

### Contextual Weight Extraction

出力側の手法。構成語フレーズをテンプレート文に埋め込んでモデルに順伝播させ、最終トークン位置の隠れ状態を出力重みベクトルとして取り出す。Tokens2Words (Kaplan+ ICLR 2025) が示した "モデルはサブワード列の最終位置に語水準の表現を構築する" という知見に基づく。複数テンプレートの平均を取ることで文脈依存のバイアスを低減する。

抽出した重みは既存語彙のロジット分布に合わせてスケーリングする。生のロジットは既存語彙の約4倍になるため、上位語彙の平均ロジットに揃えるキャリブレーションが必要になる。

### 比較手法

OMP による疎線形結合と、最適輸送 (均衡 / 非均衡 / Gromov-Wasserstein) による合成も実装している。

## 現状の結果

モデルは Pythia-410m を使用。仮想トークン間での正答率で評価している。正答率は、正解の仮想トークンが他の仮想トークンより上位にランクされた割合を指す。

| 関係タイプ | 正答率 |
|---|---|
| 地名+食品 | 98% |
| 地名+建造物 | 95% |
| 素材+人工物 | 63% |
| 用途+容器 | 33% |

87語の混合語彙での全体正答率は79%。生成タスクでは複数テンプレート平均方式で適中率100%を達成している。

素材+人工物と用途+容器の精度が低い原因は、モデルの表現空間上でこれらのカテゴリが "物理的対象" という共通の意味特徴を強く共有しており、カテゴリ間の区別が表現幾何に埋まっていないことにある。平均中心化やカテゴリ対照抽出などの事後処理を試したが改善しなかった。

## セットアップ

```
uv sync
```

## 実行

```
uv run python scripts/eval_contextual_head.py
```

結果は `results/` に JSON で出力される。スクリプトの一覧は `scripts/` を参照。

* PyTorch
* Transformers
* NumPy
* POT
